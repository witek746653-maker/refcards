*Терминал пугает только первые 10 минут. Потом ты понимаешь, что это самый прямой разговор с компьютером из возможных. Карточка разберёт: как устроена любая команда, как не убить продакшн через `rm -rf`, чем отличаются bash и PowerShell, и почему каждый Junior обязан уметь работать с CLI — причём не только теоретически.*

---

### 🎯 Зачем это знать
```
🎯 Зачем:     Управлять компьютером текстом — быстрее, точнее и мощнее, чем мышью.
⚡ Без этого: Не запустишь сервер, не установишь зависимости, не задеплоишь проект.
✅ С этим:    Git, npm/pip, Docker, SSH — всё это работает через CLI.
💼 На работе: Каждый Junior использует терминал минимум 50 раз в день. 🔥 Часто на собеседовании
```

### 🗺 Главная схема — карта системы
```
КАК РАБОТАЕТ ЛЮБАЯ КОМАНДА:
────────────────────────────────────────────────────────
  команда     флаги/опции     аргументы
  ───────     ───────────     ─────────
   git         -m              "first commit"
    ↑               ↑               ↑
  что делать  как делать    над чем делать
────────────────────────────────────────────────────────

НАВИГАЦИЯ (СТРУКТУРА ФАЙЛОВОЙ СИСТЕМЫ):
/                          ← корень (root)
├── home/
│   └── user/              ← твоя домашняя папка (~)
│       ├── projects/
│       └── .bashrc        ← файлы с точкой = скрытые
└── etc/                   ← настройки системы

Ты всегда находишься в какой-то папке — это pwd (print working dir).
Аналогия: CLI — это Проводник Windows, только ты печатаешь путь вместо кликов.
```

| Задача | Этот инструмент подходит? | Альтернатива |

### 🧩 Специальный блок (Code Matrix)

**💡 Суть:** CLI — это разговор с компьютером текстом. Ты пишешь команду → компьютер отвечает.
**🧠 Аналогия:** Представь, что компьютер — это повар. GUI — это меню с картинками. CLI — ты говоришь повару напрямую: «Принеси мне папку projects, удали файл temp.txt, создай новый».
```
⚙️ Профиль:
  Оболочки:   bash / zsh (Linux, macOS) | PowerShell / CMD (Windows)
  Версия:     bash 5+ / zsh 5+ / PowerShell 7+
  Кросс:      большинство команд ниже работают везде (различия — отдельно)
```

**🗂 НАВИГАЦИЯ — знать наизусть:**
| Команда | Что делает | PowerShell аналог |
| :--- | :--- | :--- |
| `pwd` | Где я сейчас? | `pwd` |
| `ls` / `ls -la` | Список файлов / с деталями | `dir` / `ls` |
| `cd projects` | Войти в папку | `cd projects` |
| `cd ..` | Подняться на уровень выше | `cd ..` |
| `cd ~` | Перейти домой | `cd ~` |
| `cd /` | Перейти в корень | `cd \` |

**📁 ФАЙЛЫ — знать наизусть:**
| Команда | Что делает | PowerShell аналог |
| :--- | :--- | :--- |
| `touch file.txt` | Создать пустой файл | `ni file.txt` |
| `mkdir folder` | Создать папку | `mkdir folder` |
| `cp a.txt b.txt` | Скопировать файл | `cp a.txt b.txt` |
| `mv a.txt ../` | Переместить/переименовать | `mv a.txt ../` |
| `rm file.txt` | Удалить файл ⚠️ без корзины! | `rm file.txt` |
| `rm -rf folder/` | Удалить папку рекурсивно 💀 | `rm -r folder/` |
| `cat file.txt` | Показать содержимое | `cat file.txt` |

**🔍 ПОИСК И ПОМОЩЬ:**
| Команда | Что делает |
| :--- | :--- |
| `man ls` | Документация команды (Linux/macOS) |
| `ls --help` | Краткая помощь |
| `history` | История команд |
| `grep "слово" file.txt` | Найти текст в файле |
| `find . -name "*.js"` | Найти файлы по маске |

**⚙️ ПРОЦЕССЫ И СИСТЕМА:**
| Команда | Что делает |
| :--- | :--- |
| `ps aux` / `tasklist` | Список запущенных процессов |
| `kill 1234` | Убить процесс по PID |
| `Ctrl+C` | Остановить текущий процесс |
| `Ctrl+Z` | Приостановить процесс |
| `clear` / `cls` | Очистить экран |
| `exit` | Закрыть терминал |

**🔗 ОПЕРАТОРЫ (суперсила CLI):**
| Оператор | Смысл | Пример |
| :--- | :--- | :--- |
| `&&` | Выполни следующее ТОЛЬКО если первое успешно | `npm install && npm start` |
| `\|\|` | Выполни следующее ТОЛЬКО если первое провалилось | `test \|\| echo "fail"` |
| `\|` (pipe) | Передай вывод одной команды в другую | `ls \| grep ".js"` |
| `>` | Запиши вывод в файл (перезапись) | `echo "hi" > log.txt` |
| `>>` | Добавь вывод в конец файла | `echo "bye" >> log.txt` |

![[PROG-001_scheme.svg|right|350]]
*Требуется: блок-схема структуры команды CLI: горизонтальная строка с тремя блоками «команда → флаги → аргументы», стрелка вниз к блоку «терминал», стрелка вниз к блоку «вывод результата / ошибка», два цвета для успеха (зелёный) и ошибки (красный), стиль простой, без текста на схеме кроме меток блоков.*

### 🌓 Контраст и ловушки

**❌ Частые ошибки новичка:**
```bash
# ОШИБКА 1: rm без проверки — файлы не в корзине
rm -rf /home/user/   # удалит ВСЁ в домашней папке навсегда

# ✅ Правильно: сначала ls, потом rm
ls /home/user/
rm -rf /home/user/temp/   # конкретная папка

# ОШИБКА 2: путаница относительного и абсолютного пути
cd projects/my-app   # ❌ если ты не в папке где есть projects
cd ~/projects/my-app # ✅ абсолютный путь от дома — всегда работает

# ОШИБКА 3: пробелы в именах файлов
cd my project   # ❌ воспринимает как два аргумента
cd "my project" # ✅ кавычки спасают
cd my\ project  # ✅ или экранирование
```

💬 **Запомни:** `rm` = мгновенное уничтожение. Всегда проверяй путь через `ls` перед удалением.

**Путаница: абсолютный vs относительный путь:**
```
Абсолютный → от корня, начинается с / или ~
  /home/user/projects/app   ← работает откуда угодно

Относительный → от текущей папки (pwd)
  ./projects/app  или  projects/app  ← зависит от где ты сейчас

🧪 Тест: ты в /home/user — сработает ли cd documents/readme.txt?
         Нет — это файл, не папка. cd — только для директорий.
```

**Оболочки bash vs zsh vs PowerShell:**
```
bash   → стандарт Linux-серверов, Ubuntu
zsh    → macOS по умолчанию с 2019, чуть мощнее bash
PowerShell → Windows, другой синтаксис (Get-ChildItem вместо ls)
💡 Для разработки: изучай bash/zsh — они на серверах.
```

### 🧪 Примеры в контексте

**🔨 Первый шаг — создать структуру нового проекта:**
```bash
mkdir my-app          # создаём папку проекта
cd my-app             # заходим в неё
mkdir src tests       # создаём вложенные папки
touch src/index.js    # создаём файл
touch README.md       # и ещё один
ls -la                # смотрим что получилось
```
*Доказывает: 6 команд = вся структура проекта за 5 секунд. Без GUI-кликов.*

**🏗 Реальная задача — установка и запуск Node.js проекта:**
```bash
git clone https://github.com/user/repo    # клонировать репозиторий
cd repo                                   # войти в папку
ls                                        # убедиться что есть package.json
npm install                               # установить зависимости
npm run dev                               # запустить dev-сервер
# Ctrl+C                                 # остановить когда надо
```
*Доказывает: это стандартный флоу в каждой новой команде. Знать обязательно.*

**💼 На собеседовании — работа с логами сервера:**
```bash
# "Найди все ошибки в лог-файле"
grep "ERROR" server.log            # все строки с ERROR
grep -c "ERROR" server.log         # только количество
grep "ERROR" server.log | wc -l   # то же через pipe
tail -f server.log                 # следить за логом в реальном времени
```
*Доказывает: pipe (|) — это навык, который отличает тех кто "немного знает CLI" от тех кто им владеет.*

❓ **[Своими словами]** Объясни другу (не разработчику) зачем нужен терминал, если есть удобный Проводник/Finder с иконками.

❓ **[Найди ошибку]:**
```bash
cd /home/user
mkdir new project       # создаём папку для нового проекта
cd new project
touch index.html
```
Что здесь пойдёт не так и как исправить?

❓ **[Применение]** Тебе нужно скопировать 200 файлов `.log` из папки `/var/logs` в папку `/backup`, переименовав их с префиксом `backup_`. Стал бы ты это делать мышью? Как бы ты подошёл к этой задаче через CLI?
