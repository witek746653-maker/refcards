# 🐧 Разговорник для общения с Linux
### Полное руководство для тех, кто впервые открыл терминал и немного испугался

---

## 🧠 Часть 1. Философия терминала: почему всё выглядит вот так?

Представь, что ты попал в страну, где говорят очень коротко и чётко. Никаких «будьте добры, не могли бы вы...». Только: **«Принеси. Хлеб. Быстро.»** Именно так работает терминал.

Когда ты кликаешь мышкой по красивым кнопкам — ты общаешься с компьютером через переводчика. Этот переводчик удобный, но медленный и ограниченный. Терминал — это прямой разговор **без посредников**.

### Команды — это глаголы

Любая команда — это **глагол действия**. Что ты хочешь *сделать*?

- `copy` — скопировать
- `move` — переместить  
- `list` — перечислить

Флаги и аргументы — это **наречия и дополнения**, которые уточняют *как* и *что*:

> *«Перечисли (ls) — подробно (-l) — всё, включая скрытое (-a) — вот эту папку (/home/user)»*

Звучит странно только поначалу. Через неделю практики ты будешь читать команды как обычные предложения. Обещаю.

## 📐 Часть 2. Грамматика команд: формула, которая объясняет всё

```
КОМАНДА   [ФЛАГИ]   [ОБЪЕКТ]
  ↓           ↓         ↓
 Глагол    Наречие   Существительное
```

### Живой пример:

```bash
ls   -la   /home/username
```
- `ls` — «покажи список»
- `-la` — «подробно (`l`) и вместе со скрытыми файлами (`a`)»
- `/home/username` — «вот в этой папке»

---

### Один дефис vs два дефиса — в чём разница?

| Запись | Смысл | Пример |
|--------|-------|--------|
| `-l` | Короткий флаг, одна буква | `ls -l` |
| `--long` | Полное слово, для читаемости | `ls --all` |
| `-la` | Несколько коротких флагов слеплены вместе | = `-l -a` |

> 💡 **Правило**: `-l` и `--long` — это часто одно и то же, просто разный стиль написания.

---

### Пробел — это святое ⚠️

В Bash пробел — разделитель слов. Это не украшение, это **часть грамматики**.

```bash
# ✅ Правильно — три отдельных слова
cp -r папка1 папка2

# ❌ Неправильно — команда не поймёт
cp-r папка1 папка2
```

Если в имени файла есть пробел — оберни в кавычки:
```bash
cp "мой документ.txt" backup/
```

## 📖 Часть 3. Словарь-переводчик: ТОП-15 команд

| Команда | Расшифровка | Что делает | Пример |
|---------|-------------|------------|--------|
| `ls` | **L**i**s**t | Показывает содержимое папки | `ls -la` |
| `cd` | **C**hange **D**irectory | Переходит в другую папку | `cd /home/user` |
| `pwd` | **P**rint **W**orking **D**irectory | Говорит, где ты сейчас | `pwd` |
| `mkdir` | **M**a**k**e **Dir**ectory | Создаёт новую папку | `mkdir проект` |
| `touch` | touch (прикоснуться) | Создаёт пустой файл | `touch notes.txt` |
| `cp` | **C**o**p**y | Копирует файл или папку | `cp file.txt backup/` |
| `mv` | **M**o**v**e | Перемещает или переименовывает | `mv old.txt new.txt` |
| `rm` | **R**e**m**ove | Удаляет файл (без корзины!) | `rm file.txt` |
| `cat` | con**cat**enate | Выводит содержимое файла | `cat notes.txt` |
| `echo` | echo (эхо) | Выводит текст на экран | `echo "Привет!"` |
| `grep` | **G**lobal **R**egular **E**xpression **P**rint | Ищет текст внутри файлов | `grep "ошибка" log.txt` |
| `find` | find (найти) | Ищет файлы по системе | `find . -name "*.jpg"` |
| `chmod` | **Ch**ange **mod**e | Меняет права доступа | `chmod +x script.sh` |
| `sudo` | **S**uper**u**ser **do** | Выполняет от имени администратора | `sudo apt update` |
| `history` | history (история) | Показывает список последних команд | `history` |

> 🎯 **Лайфхак**: нажми стрелку `↑` на клавиатуре — и увидишь предыдущую команду. Можно листать всю историю!

## 🔧 Часть 4. Конструктор фраз: логические цепочки

Вот где начинается настоящая магия. Представь команды как **кубики Lego**: каждую можно соединить с другой, и результат будет больше суммы частей.

---

### 🚿 Труба (Pipe) `|` — водопровод для данных

**Идея:** результат работы одной команды становится **входящими данными** для следующей.

```
[Команда 1] → вывод → | → становится входом → [Команда 2]
```

**Аналогия с водопроводом:** первая труба наполняет ведро водой, вторая труба фильтрует эту воду, третья — разливает по стаканам. Каждый этап получает результат предыдущего.

```bash
# Покажи список файлов, но только те строки, где есть слово "photo"
ls -la | grep "photo"

# Посчитай, сколько процессов запущено
ps aux | wc -l

# Покажи 10 самых больших файлов в папке
du -sh * | sort -rh | head -10
```

> Читай `|` как слово **«затем»** или **«и передай это»**.

---

### 💾 Перенаправление `>` и `>>` — сохрани мысль в файл

Обычно команды выводят результат **на экран**. Но ты можешь сказать: «нет, запиши это в файл».

| Символ | Действие | Аналогия |
|--------|----------|----------|
| `>` | Записать в файл (перезаписывает!) | Чистый лист бумаги — старое стирается |
| `>>` | Добавить в конец файла | Продолжаешь писать на том же листе |

```bash
# Сохранить список файлов в файл (перезапишет)
ls -la > список_файлов.txt

# Добавить строчку в конец лога
echo "Запуск скрипта: $(date)" >> app.log

# Сохранить ошибки отдельно
команда 2> errors.log
```

> ⚠️ **Осторожно с `>`!** Он безжалостно уничтожает содержимое файла. Используй `>>`, если не уверен.

---

### 🔍 grep — детектив внутри текста

`grep` — это как Ctrl+F в браузере, только мощнее в сотни раз.

```bash
# Найти слово "error" в файле лога
grep "error" server.log

# Найти без учёта регистра (-i)
grep -i "ERROR" server.log

# Показать номера строк (-n)
grep -n "error" server.log

# Найти всё, что НЕ содержит слово (-v)
grep -v "debug" server.log

# Рекурсивный поиск во всех файлах папки (-r)
grep -r "TODO" ./src/
```

**Комбо с трубой:**
```bash
# Покажи запущенные процессы, но только nginx
ps aux | grep nginx

# Найди все ошибки за сегодня
cat server.log | grep "$(date +%Y-%m-%d)" | grep "error"
```

## 🧩 Часть 5. Как «думать на Linux»: алгоритм из 3 шагов

Допустим, задача звучит по-человечески:  
> *«Найти все картинки формата .jpg в папке проекта и сохранить их список в файл»*

### Шаг 1: Разбей задачу на действия

- Найти файлы → `find`
- Отфильтровать по расширению .jpg → флаг `-name "*.jpg"`
- Сохранить результат в файл → `>`

### Шаг 2: Найди команду для каждого действия

```
найти файлы    →   find ./проект -name "*.jpg"
сохранить      →   > список_картинок.txt
```

### Шаг 3: Собери цепочку

```bash
find ./проект -name "*.jpg" > список_картинок.txt
```

**Ещё пример — задача посложнее:**
> *«Найти все строки с ошибками в логе, отсортировать и показать только уникальные»*

```
1. Прочитать лог        → cat server.log
2. Найти строки с ERROR → grep "ERROR"
3. Отсортировать        → sort
4. Убрать дубликаты     → uniq
```

```bash
cat server.log | grep "ERROR" | sort | uniq
```

## ⚗️ Часть 6. Практические заклинания с разбором

### Заклинание 1: «Найди и посчитай»

```bash
grep -r "TODO" ./src/ | wc -l
```

| Кусок | Смысл |
|-------|-------|
| `grep -r "TODO"` | Ищи слово TODO рекурсивно |
| `./src/` | Вот в этой папке |
| `\|` | И передай результат дальше |
| `wc -l` | Посчитай количество строк (= количество совпадений) |

**Результат:** число в терминале — сколько TODO осталось в коде.

---

### Заклинание 2: «Архивируй проект»

```bash
tar -czf backup_$(date +%Y%m%d).tar.gz ./my_project/
```

| Кусок | Смысл |
|-------|-------|
| `tar` | Архиватор |
| `-c` | Создать архив |
| `-z` | Сжать через gzip |
| `-f` | Следующий аргумент — имя файла |
| `backup_$(date +%Y%m%d)` | Имя с датой: например `backup_20250615` |
| `.tar.gz` | Расширение |
| `./my_project/` | Что архивируем |

---

### Заклинание 3: «Покажи самые большие файлы»

```bash
du -sh /var/log/* | sort -rh | head -5
```

| Кусок | Смысл |
|-------|-------|
| `du -sh` | Покажи размер каждого объекта |
| `/var/log/*` | Всё, что внутри /var/log |
| `sort -rh` | Отсортируй по размеру, крупные — вверху |
| `head -5` | Оставь только первые 5 строк |

---

### Заклинание 4: «Найди и замени во всех файлах»

```bash
find . -name "*.txt" -exec sed -i 's/старое/новое/g' {} \;
```

| Кусок | Смысл |
|-------|-------|
| `find . -name "*.txt"` | Найди все .txt файлы |
| `-exec` | И для каждого выполни... |
| `sed -i` | Замени прямо в файле |
| `'s/старое/новое/g'` | Найди «старое», замени на «новое», везде (g) |
| `{} \;` | `{}` — это место для имени файла, `\;` — конец команды |

---

### Заклинание 5: «Мониторинг в реальном времени»

```bash
tail -f /var/log/syslog | grep --line-buffered "error"
```

| Кусок | Смысл |
|-------|-------|
| `tail -f` | Следи за файлом в реальном времени (как стриминг) |
| `/var/log/syslog` | Системный лог |
| `grep --line-buffered "error"` | Показывай только строки с "error", не тормози поток |

**Результат:** живой фильтр ошибок. Очень полезно при отладке.

## 🛡️ Часть 7. Шпаргалка по выживанию

### Как не сломать систему

| Правило | Почему |
|---------|--------|
| ❌ Никогда не пиши `rm -rf /` | Удалит **всю систему** без возможности восстановления |
| ⚠️ Будь осторожен с `sudo` | Ты становишься всемогущим — любая ошибка критична |
| ✅ Сначала проверяй без действия | `find ... -name "*.log"` перед `find ... -delete` |
| ✅ Делай бэкапы | `cp важный_файл важный_файл.bak` перед правкой |

---

### Твои лучшие друзья: `man` и `--help`

Забыл, что делает команда? Не гугли — спроси саму систему!

```bash
# Полное руководство (выход — клавиша q)
man ls

# Краткая справка
ls --help

# Быстрый поиск команды по описанию
apropos "copy files"
```

> 💡 В `man` используй `/слово` для поиска и `n` для перехода к следующему совпадению.

---

### Что делать, если всё зависло

| Ситуация | Решение |
|----------|---------|
| Команда не отвечает | `Ctrl + C` — прервать процесс |
| Набрал что-то лишнее | `Ctrl + U` — очистить строку |
| Хочешь выйти из терминала | `exit` или `Ctrl + D` |
| Зависла программа намертво | `Ctrl + Z` — приостановить, потом `kill %1` |
| Всё совсем плохо 😱 | `Ctrl + Alt + T` — новый терминал, `top` — найти виновника |

---

### Секретные суперспособности терминала

```bash
# Автодополнение — твой лучший друг
cd /ho[TAB]        # нажми Tab — дополнит до /home/

# Поиск в истории команд
Ctrl + R           # начни печатать — найдёт похожую команду из прошлого

# Быстрый переход домой
cd ~               # всегда ведёт в /home/твой_пользователь/

# Предыдущая папка
cd -               # вернуться туда, где был только что
```

## 🎓 Послесловие

Терминал — это не страшный чёрный экран из фильмов про хакеров. Это **самый честный инструмент**: он делает ровно то, что ты говоришь, без скрытых сюрпризов.

Первые дни будет казаться, что ты учишь иностранный язык. Это нормально — ты **действительно** учишь иностранный язык. Но уже через месяц практики руки сами будут тянуться к терминалу вместо мышки, и ты поймёшь: **это быстрее, точнее и красивее**.

> *«Дай мне достаточно pipe-ов, и я переверну весь интернет»*  
> — каждый Unix-разработчик после первой недели

Удачи. Ты справишься. `echo "Ты молодец"` 🐧
