### 1. ВЫБОР КОНТЕНТА [ОБЯЗАТЕЛЬНО]

- **80/20:** Выбирай только те 20% знаний о концепции, которые покрывают 80% реальных задач начинающего. Всё, что не нужно для первой работы — убирай.
- **Приоритет:** Сначала объясни **зачем это нужно** (мотивация), потом **как работает** (механика), потом **как написать** (код).
- **Язык:** Каждый термин на английском — сразу с переводом. Пример: `function (функция)`, `variable (переменная)`.
- **Актуальность:** Только современный синтаксис. Устаревшие подходы — только в блоке «Ловушки», с пометкой `⚠️ Устарело`.
- **Сложность кода:** Максимально простые примеры. Никакого кода, который новичок не поймёт без отдельного объяснения.
- **🎯 Job-фильтр:** Для каждой концепции обязательно укажи — спрашивают ли это на собеседовании Junior:
  - `🔥 Часто на собеседовании` — знать обязательно
  - `📋 Иногда спрашивают` — желательно знать
  - `💤 Редко для Junior` — можно пропустить пока

### 2. МЕТАДАННЫЕ КАРТОЧКИ [ОБЯЗАТЕЛЬНО]

> **ВАЖНО!** Сама таблица метаданных (со шкалами `🎯 Актуальность`, `🧠 Сложность` и т.д.) берётся из `Core_Template.md`. Ты обязан скопировать её в начало ответа КАК ЕСТЬ и заполнить значениями, ничего не удаляя.

Для `{{KIND}}` используй ТОЛЬКО:
- `Concept` — фундаментальная идея (переменная, функция, цикл, условие)
- `Pattern` — паттерн (шаблон решения повторяющейся задачи)
- `Syntax` — конкретная конструкция языка (как написать цикл, как объявить функцию)
- `Tool` — инструмент или библиотека (Git, npm, React)
- `Principle` — принцип разработки (DRY — не повторяйся, KISS — делай проще)

Для `{{LEVEL}}` используй ТОЛЬКО:
- `Foundations` — самый старт (переменные, типы, циклы, функции)
- `Junior` — следующий уровень (массивы, объекты, DOM, запросы к серверу)

> ⚠️ Карточки уровня `Middle` и `Senior` для этого пользователя не создаются.

### 5. БЛОК {{DOMAIN_CONTENT}} (Code Matrix) [ОБЯЗАТЕЛЬНО]

**5.1. Core Insight [ОБЯЗАТЕЛЬНО]**
Одно предложение, которое объясняет суть — так, чтобы понял человек без опыта.
Формат: `💡 Суть: [объяснение без жаргона]`

Обязательно добавь бытовую аналогию:
`🧠 Аналогия: [сравнение с чем-то из жизни — не из IT]`

✅ Хорошая аналогия: *Переменная — это коробка с подписью. Кладёшь внутрь что угодно и находишь по названию.*
❌ Плохая аналогия: *Переменная — это как ячейка памяти.* ← новичок не знает что такое «ячейка памяти».

**5.2. Технический минимум [ОБЯЗАТЕЛЬНО]**
```
⚙️ Профиль:
  Язык примеров:   [JavaScript / Python / язык-нейтрально]
  Термин EN:       [английское название] = [русский перевод]
  Версия:          [если важна: ES2020+ / Python 3+ / иначе — не указывай]
```

**5.3. Когда использовать [ОБЯЗАТЕЛЬНО]**
| ✅ Используй | ❌ Не используй |
| :--- | :--- |
| [простая ситуация] | [простая ситуация] |

Минимум 3 строки. Формулируй через ситуации, не через технические условия.
Пример: ✅ «Когда нужно повторить одно действие много раз» — не «Когда итерируешь коллекцию».

**5.4. Code Matrix [ОБЯЗАТЕЛЬНО]**
Минимальный рабочий пример на языке из метаданных.

```
// [одна строка: что делает этот код — по-русски]
[код]
```

Требования:
- Максимум 10–12 строк
- Комментарий `//` к КАЖДОЙ строке (новичок читает построчно)
- Называй переменные по-русски транслитом или по смыслу (`imya`, `vozrast`, `spisok`) — не `x`, `foo`, `temp`
- Никаких импортов и конфигов если не критично

**5.5. Визуализация [ОБЯЗАТЕЛЬНО]**

```
![[{{ID}}_scheme.svg|right|350]]
```

ТЗ строго в формате:
```
*Требуется: [тип: блок-схема / простая схема] + [что изображено словами] + [ключевые шаги/блоки] + [стиль: простой, понятный новичку, без лишних деталей]*
```

✅ Пример: *Требуется: блок-схема, как работает условие if/else: ромб «условие» → две стрелки «да» и «нет» → два блока действий, стиль: простой, цветные стрелки.*
❌ Запрещено: *Требуется: схема концепции.*

### 6. БЛОК {{CONTRAST_CONTENT}} [ОБЯЗАТЕЛЬНО]

**6.1. Частая ошибка новичка [ОБЯЗАТЕЛЬНО]**
Самая распространённая ошибка при первом знакомстве с этой концепцией.
Формат:
```
❌ Как делают новички:
[код с ошибкой — 3–5 строк, с комментарием что здесь не так]

✅ Как правильно:
[исправленный код — 3–5 строк]

💬 Запомни: [одно простое правило одной строкой]
```

**6.2. Путаница с похожей концепцией [ОБЯЗАТЕЛЬНО если есть пара]**
Две концепции, которые новички постоянно путают.
Формат:
```
[Концепция A] vs [Концепция B]:
A → [простое объяснение + когда]
B → [простое объяснение + когда]
🧪 Тест: [один простой вопрос, который помогает не запутаться]
```

Примеры пар для новичка: `let vs const`, `== vs ===`, `function vs arrow function`, `array vs object`.

### 7. БЛОК {{EXAMPLES_CONTENT}} [ОБЯЗАТЕЛЬНО]

Ровно 3 примера. Никаких `foo` / `bar` / абстрактных `MyClass`.

`🔨 Первый шаг` — самый простой пример, только механика. Как будто объясняешь другу.
`🏗 Реальная задача` — применение в задаче, с которой Junior столкнётся на работе (список товаров, форма, кнопка, запрос к API).
`💼 На собеседовании` — пример или вопрос в том виде, в каком его задают на интервью для Junior.

Под каждым примером:
- Язык указан в метке кода
- Каждая строка кода — с комментарием `//`
- Одна строка итога: **что этот пример доказывает**

### 8. БЛОК {{QUIZ_CONTENT}} [ОБЯЗАТЕЛЬНО]

Ровно 3 вопроса. Цель — убедиться что человек понял, а не запомнил.

Используй только эти типы:
- `❓ [Своими словами]` — объясни концепцию так, как объяснил бы другу (проверяет понимание, не память).
- `❓ [Найди ошибку]` — покажи короткий код с одной ошибкой, попроси найти и исправить.
- `❓ [Применение]` — опиши задачу из жизни, спроси использовал бы эту концепцию и почему.

> ⚠️ Запрещено: вопросы типа «дай определение», «перечисли свойства», «назови метод». Это зубрёжка, а не понимание.
