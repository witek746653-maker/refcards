# **RefCards — Фронтенд**

## **Стек и архитектура**

* **Vanilla JS \+ HTML/CSS.** Никаких тяжелых фреймворков (React/Vue отсутствуют). Вся логика инкапсулирована в frontend/index.html и смежных файлах.  
* **Инструментарий:** В package.json присутствует Vite/ESBuild, но они используются преимущественно для локального dev-сервера или базового бандлинга, исходный код пишется на нативном JS.  
* **Внешние библиотеки (CDN):** \- marked.js — рендеринг Markdown в HTML.  
  * CodeMirror — встроенный редактор кода для карточек.

## **Взаимодействие с API**

* Фронтенд не имеет жестко захардкоженного URL бэкенда. URL задается пользователем через UI и сохраняется в localStorage.  
* Данные (список карточек, контент) запрашиваются напрямую у Flask API (ручки /api/\*).

## **Рендеринг (Важно\!)**

Внутри index.html реализована специфичная логика wrapSections. Она ожидает определенную структуру DOM после парсинга Markdown (например, цепочку тегов h1 → p → hr → table). Это необходимо для корректного оборачивания заголовков и метаданных в UI-компоненты. При изменении логики рендеринга нужно следить, чтобы не сломать эти паттерны.
