# **RefCards — Данные и Карточки**

## **Source of Truth**

Основной источник данных — файлы \*.md в директории cards/. База SQLite на бэкенде служит кэшем/индексом для быстрого поиска и API, но финальное состояние всегда должно отражаться в файлах.

## **Спецификация файла карточки**

* **Именование:** \[Домен\]-Название.md (например, \[LYR\]-comme-ils-disent.md).  
* **Модули (домены):** Описаны в Module\_\*.md (Art, Literature, Programming и т.д.). Каждая карточка принадлежит одному из модулей.

## **Структура файла (Core\_Template)**

Строгих линтер-инвариантов нет, но фронтенд и бэкенд (check\_content.py) ожидают следующий паттерн:

1. **YAML Frontmatter:** В самом начале файла между \---. Должен содержать как минимум ключи name и domain.  
2. **Заголовок:** h1 (\# Заголовок).  
3. **Описание:** Абзац текста (p).  
4. **Разделитель:** \--- (hr).  
5. **Метаданные (Таблица):** Ожидается table с дополнительными характеристиками.

*Вывод:* Нарушение этой последовательности не сломает бэкенд, но может сломать красивое отображение карточки на фронтенде из\-за особенностей работы функции wrapSections.
