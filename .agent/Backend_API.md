# **RefCards — Бэкенд и API**

## **Стек**

* **Фреймворк:** Flask 3  
* **БД:** SQLite  
* **CORS:** Включен (позволяет фронтенду стучаться с других доменов).

## **Роль бэкенда**

Бэкенд выступает прослойкой между SQLite и файловой системой (директория cards/). Он предоставляет CRUD-интерфейс и обслуживает логику синхронизации.

## **Основные эндпоинты (server.py)**

* GET/POST/PUT/DELETE /api/cards — классический CRUD для карточек.  
* GET /api/data/diff — вычисляет разницу между состоянием SQLite и реальными .md файлами в cards/.  
* POST /api/data/apply — применяет изменения (синхронизирует БД и файловую систему).  
* Загрузка файлов и функции экспорта.

## **Утилиты (backend/\*.py)**

* check\_content.py — парсер/экстрактор данных (читает frontmatter и контент).  
* fix.py — утилита для автоматического форматирования .md файлов.  
* find\_\*.py — скрипты локального поиска.

## **Ограничения и риски**

* **Синхронизация (Race condition):** Состояние в SQLite и в файлах .md может разъехаться, если править базу в обход /api/data/apply или править файлы без последующего обновления БД.  
* **Зависимости:** Устанавливаются через pip3 install \-r backend/requirements.txt.
